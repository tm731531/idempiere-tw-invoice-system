## 1. 系統概覽

本系統管理台灣統一發票（統一發票）的完整生命週期：

```
財政部核配字軌
      │
      ▼
① 建立字軌記錄（發票字軌管理）
      │
      ▼ （開立發票時自動建立）
② 發票號碼對應（發票對應管理）
      │
      ▼ （如有退貨/折讓）
③ 折讓記錄（進項折讓管理）
      │
      ▼ （雙月期末）
④ 401申報表產生（401申報表管理）
      │
      ▼
匯出電子申報檔 → 上傳財政部
```

**申報週期**：雙月制
| 期別 | 月份 | 申報截止 |
|------|------|---------|
| 第1期 | 1–2月 | 3月15日 |
| 第2期 | 3–4月 | 5月15日 |
| 第3期 | 5–6月 | 7月15日 |
| 第4期 | 7–8月 | 9月15日 |
| 第5期 | 9–10月 | 11月15日 |
| 第6期 | 11–12月 | 隔年1月15日 |

## 2. 操作流程總覽

### 每期作業標準流程

```
期初（收到財政部字軌核配通知）
  └─ 步驟1：在「發票字軌管理」建立新字軌 → 狀態改為「使用中」

期中（日常開立發票）
  └─ 在 iDempiere 正常開立 C_Invoice
  └─ 步驟2：系統自動建立「發票對應管理」記錄
  └─     B2B 三聯式：確認買方統一編號已填入
  └─ 步驟3（若有退貨/折讓）：在「進項折讓管理」建立折讓記錄

期末（雙月結束後，申報截止前）
  └─ 步驟4：執行「產生401申報表」流程
  └─ 步驟5：在「401申報表管理」確認並補充數字
  └─ 步驟6：執行「匯出電子申報」→ 上傳財政部
```

## 3. 視窗一：發票字軌管理

**路徑**：選單 → Taiwan Invoice Prefix 發票字軌

**用途**：記錄財政部每兩個月核配給公司的發票字軌（如 AA、AB 等），管理號碼範圍與使用狀態。

### 3.1 欄位說明

| 欄位 | 說明 | 必填 | 備註 |
|------|------|------|------|
| 廠商 | 所屬公司（自動填入） | ✓ | 不可修改 |
| 組織 | 所屬組織/分公司 | ✓ | 下拉選擇 |
| 啟用 | 此記錄是否有效 | | 預設勾選 |
| **字軌代號** | 財政部核配的2碼英文字母 | ✓ | 如 AA、AB、AK，不含數字 |
| **發票類型** | 三聯式 / 二聯式 | ✓ | B2B選三聯式；B2C選二聯式 |
| **狀態** | 字軌目前狀態 | ✓ | 見下方狀態說明 |
| **字軌起始日期** | 財政部核配有效期間開始 | ✓ | 通常為雙月第一天 |
| **字軌結束日期** | 財政部核配有效期間結束 | ✓ | 通常為雙月最後一天 |
| **起始號碼** | 本批字軌的起始8位編號 | ✓ | 如 01234567 |
| **結束號碼** | 本批字軌的結束8位編號 | ✓ | 如 01300000 |
| 目前號碼 | 目前已開立到的號碼 | | 系統自動更新，勿手動修改 |
| 最後發票號碼 | 最後一張發票的完整號碼 | | 系統自動更新 |
| 最後開立日期 | 最後開立日期 | | 系統自動更新 |

### 3.2 狀態流程

```
I（未啟用/Inactive）
    │ 手動改為 A
    ▼
A（使用中/Active）
    │ 號碼用完時系統改為 C
    ▼
C（已用完/Complete）── 不可逆，不可改回 A
```

> **注意**：C 狀態代表字軌已全數開完，不得再開立發票。系統會在最後一個號碼開出後自動切換。

> **狀態轉換限制（台灣稅法）**：狀態只允許單向前進：I → A → C。系統強制執行以下規則：
> - **不可從 A 降回 I**：字軌一旦啟用（A），即不得再降回未啟用（I）。台灣稅法要求已啟用字軌須持續追蹤，不得任意停用。
> - **C 不可逆**：已用完（C）為終態，不可改回 A 或 I。

### 3.3 操作步驟

**建立新字軌：**
1. 點選工具列「＋」（新增）
2. 選擇「組織」
3. 填入「字軌代號」（2碼大寫英文，依財政部核配通知）
4. 選擇「發票類型」
5. 狀態先設為「I（未啟用）」
6. 填入「字軌起始/結束日期」（依財政部核配期間）
7. 填入「起始號碼」和「結束號碼」（依財政部核配通知）
8. 儲存後，再將狀態改為「A（使用中）」

> 同一組織同時只能有一個 A 狀態的同類型字軌。

## 4. 視窗二：發票對應管理

**路徑**：選單 → Taiwan Invoice Prefix Map 發票對應

**用途**：記錄每張 iDempiere 發票（C_Invoice）對應的台灣字軌號碼。通常由系統在開立發票時自動建立，**一般不需手動操作**。

### 4.1 欄位說明

| 欄位 | 說明 | 必填 | 備註 |
|------|------|------|------|
| 廠商 | 所屬公司 | ✓ | 自動填入 |
| 組織 | 所屬組織 | ✓ | 自動填入 |
| 啟用 | 此記錄是否有效 | | |
| **發票字軌** | 對應的字軌記錄 | ✓ | 連結至「發票字軌管理」 |
| **發票** | iDempiere 原始發票 | ✓ | C_Invoice，下拉或搜尋選取 |
| **發票日期** | 發票開立日期 | ✓ | 必須在字軌有效期內 |
| **發票號碼** | 完整台灣發票號碼 | ✓ | 格式：字軌2碼 + 8位數字，如 AA01234567 |
| 買方統一編號 | 買方統一編號（8位數字） | △ | **三聯式（B2B）法定必填**；二聯式可留空 |
| 到期警示 | 字軌即將到期警示旗標 | | 系統自動勾選，距到期90天內觸發 |

### 4.2 買方統一編號規則

- **三聯式（B2B）**：法律規定必填，格式為8位純數字（如 `12345678`）
- **二聯式（B2C）**：買方為一般消費者，此欄留空
- 格式驗證：系統檢查是否為8位純數字，不符合時顯示警告

### 4.3 查詢用途

此視窗主要用於：
- **查詢**某張發票的台灣發票號碼
- **核對**字軌使用情況（已開幾號）
- **追蹤**到期警示（IsExpiryWarning=Y 表示即將到期）

## 5. 視窗三：進項折讓管理

**路徑**：選單 → Taiwan Invoice Adjustment 進項折讓

**用途**：記錄進項與銷項的折讓（退貨、折扣讓步等），確保在正確的申報期別申報。

### 5.1 欄位說明

| 欄位 | 說明 | 必填 | 備註 |
|------|------|------|------|
| 廠商 | 所屬公司 | ✓ | 自動填入 |
| 組織 | 所屬組織 | ✓ | |
| 啟用 | 此記錄是否有效 | | |
| 原始發票 | 被折讓的原始 iDempiere 發票 | | 選取 C_Invoice |
| **折讓類型** | RETURN（銷貨退回）或 ALLOWANCE（折讓） | ✓ | |
| **折讓方向** | SALES 或 PURCHASE | ✓ | 見下方說明 |
| **折讓日期** | 折讓發生日期 | ✓ | 不可為未來日期 |
| **折讓稅額** | 折讓的稅額金額（元） | ✓ | 只填稅額，不含本金 |
| **申報期別** | 申報的雙月期別（1–6期） | ✓ | 預設為折讓日期所屬期別 |
| 超期申報 | 是否超出原折讓所屬期別 | | 超期申報需勾選並確認風險 |

### 5.2 折讓方向說明

> **注意**：「折讓方向」（AdjustmentDirection）為必填欄位，儲存時若未選擇會被系統攔截。請務必在儲存前確認已選擇正確方向。

| 折讓方向 | 意義 | 誰開立折讓單 | 影響 |
|---------|------|------------|------|
| **SALES（銷項）** | 我方開立折讓單給買方 | 本公司 | 減少銷項稅額 |
| **PURCHASE（進項）** | 供應商開立折讓單給我方 | 供應商 | 減少可扣抵進項稅 |

### 5.3 申報期別規則

- **原則**：折讓應在發生的雙月期別申報
- **超期**：若超過當期（如1月的折讓在3月才申報），必須勾選「超期申報」
  - 超期申報可能產生**補稅及滯納金**風險，系統會顯示確認警告
  - 建議諮詢稅務顧問後再操作

### 5.4 操作步驟

1. 點選「＋」新增
2. 選擇「原始發票」（可留空，如找不到對應發票）
3. 選擇「折讓類型」
4. 選擇「折讓方向」（是我方開立，還是供應商開立給我們）
5. 填入「折讓日期」
6. 填入「折讓稅額」（元，只填稅額部分）
7. 確認「申報期別」是否正確
8. 若是超期申報，勾選「超期申報」並確認已了解風險
9. 儲存

## 6. 視窗四：401申報表管理

**路徑**：選單 → Taiwan Tax Statement 401申報表

**用途**：管理每期的營業稅申報數字（Form 401），可由系統自動產生或手動填寫，最終匯出財政部格式的電子申報檔。

### 6.1 欄位說明

> **欄位驗證範圍**：
> - **申報年度**（StatementYear）：必須介於 2000–2099 之間（西元年）。
> - **申報期別**（StatementPeriod）：必須為 1–6 的整數，超出範圍系統會拒絕儲存。

| 欄位 | 說明 | 必填 | 備註 |
|------|------|------|------|
| 廠商 | 所屬公司 | ✓ | 自動填入 |
| 組織 | 所屬組織 | ✓ | |
| 啟用 | 此記錄是否有效 | | |
| **申報年度** | 民國年度 | ✓ | 如 113（民國113年），系統儲存為西元年（2000–2099）|
| **申報期別** | 第幾期（1–6） | ✓ | 1=1–2月，2=3–4月，依此類推 |
| **應稅銷售額** | 適用5%稅率的銷售額 | ✓ | 不含稅，不含零稅率 |
| **零稅率銷售額** | 出口等零稅率銷售額 | ✓ | 可申請退稅，非免稅 |
| **免稅銷售額** | 依法免徵營業稅的銷售額 | ✓ | 不可退稅 |
| **銷項稅額** | 應稅銷售額 × 5% | ✓ | 系統計算，可調整 |
| **進項稅額** | 本期可扣抵進項稅額 | ✓ | 含折讓調整後 |
| **不可扣抵進項稅額** | 依法不可扣抵的進項稅 | ✓ | 如招待費、個人消費 |
| **上期留抵稅額** | 前期累積未用完的留抵 | ✓ | 跨期帶入，第1期填0 |
| 兼營業者 | 是否為兼營（有免稅銷售）事業 | | 勾選後啟用比例計算 |
| 應稅比率 | 應稅銷售 / 總銷售（FLOOR） | △ | 兼營業者必填 |
| 調整後進項稅額 | 按比例調整後的可扣抵進項稅 | | 系統計算 |
| **應納稅額** | 最終應繳（正值）或退稅（負值）金額 | | 系統計算，見公式 |
| 申報截止日 | 財政部規定申報截止日 | | 系統自動填入 |

### 6.2 應納稅額計算公式

```
應納稅額 = 銷項稅額
         − (進項稅額 − 不可扣抵進項稅額)
         − 上期留抵稅額

若結果 > 0：應繳稅
若結果 < 0：可申請留抵下期或退稅
```

**兼營業者調整：**
```
調整後進項稅額 = 進項稅額 × 應稅比率（FLOOR）
應納稅額 = 銷項稅額 − (調整後進項稅額 − 不可扣抵進項稅額) − 上期留抵
```

### 6.3 操作步驟

**方法一：自動產生（計劃中功能，尚未實作）**

> ⚠️ **「產生401申報表」（GenerateTaxStatementProcess）目前為計劃中功能，尚未實作。** 執行此流程將顯示「不支援」錯誤。請改用方法二手動建立。

1. 執行流程「產生401申報表」（GenerateTaxStatementProcess）
2. 選擇年度與期別
3. 系統自動彙總當期發票資料並建立申報表
4. 進入「401申報表管理」核對數字
5. 必要時手動調整特殊項目

**方法二：手動建立**
1. 點選「＋」新增
2. 填入「申報年度」（民國年）
3. 填入「申報期別」（1–6）
4. 依財政部規定逐欄填入金額
5. 確認「應納稅額」計算正確

**匯出申報檔（計劃中功能，尚未實作）：**

> ⚠️ **「匯出電子申報」（ExportTaxReportProcess）目前為計劃中功能，尚未實作。** 執行此流程將顯示「不支援」錯誤。電子申報檔須待此功能完成後方可使用。

1. 開啟對應的申報表記錄
2. 執行流程「匯出電子申報」（ExportTaxReportProcess）
3. 取得 CSV 檔案
4. 登入財政部電子申報繳稅服務網上傳

### 6.4 兼營業者注意事項

若公司同時有應稅和免稅銷售（例如：同時賣電子產品（應稅）和書籍（免稅））：

1. 勾選「兼營業者」
2. 計算「應稅比率」= 應稅銷售額 ÷ (應稅 + 免稅 + 零稅率) 總銷售額，FLOOR 取整
3. 系統自動計算「調整後進項稅額」
4. 若公司成立未滿9個月，**免做比例調整**（應稅比率填100%）

## 7. 常見問題

**Q: 欄位都是灰色無法編輯？**
A: 請重新登入 iDempiere。Bundle 安裝後新權限需重登才生效。若重登後仍全灰，確認 bundle 版本為 1.0.10（OSGi console: `ss tw.idempiere`），舊版缺少必要欄位定義會導致此問題。

**Q: 找不到 TW 相關選單？**
A: 確認 bundle 狀態為 ACTIVE（OSGi console: `ss tw.idempiere`）。若狀態異常，執行 `start <bundle_id>`。

**Q: 字軌狀態改成 C 後不能再改？**
A: 正確。C（已用完）是終態，代表字軌號碼已全數開完。請向財政部申請新的字軌。

**Q: 三聯式發票忘記填買方統一編號？**
A: 進入「發票對應管理」找到對應記錄，補填「買方統一編號」（8位純數字）。若已申報則需以更正申報方式處理。

**Q: 折讓超過當期才發現？**
A: 進入「進項折讓管理」，建立折讓記錄，勾選「超期申報」。**超期申報可能產生補稅及滯納金，建議先諮詢稅務顧問。**

**Q: 上期留抵如何帶入？**
A: 在「401申報表管理」的「上期留抵稅額」欄填入前期申報表中的留抵稅額。第一期填 0。

**Q: 零稅率和免稅的差異？**
A: 零稅率（ZeroRate）適用出口銷售，稅率為0%但**可申請退稅**進項稅；免稅（Exempt）依法不課稅且**不可退稅**進項稅。兩者都需分開填報。

**Q: 按「新增」出現「Index: 2」錯誤？**
A: 這是舊版 2Pack（1.0.9 以前）缺少 SeqNoGrid 設定造成的。請確認已部署 v1.0.10（含 2Pack 1.0.10）的 JAR，並執行**登出 → 登入**（或左側選單點「Cache Reset」）清除 metadata 緩存後再試。

**Q: 更新 bundle 後視窗行為沒有改變？**
A: iDempiere 有 dictionary metadata 緩存。更新 bundle 並確認 2Pack 安裝成功後，必須執行以下其中一項才能生效：
- 左側選單 → **Cache Reset**（立即清除，不需重啟）
- 或**登出再登入**
