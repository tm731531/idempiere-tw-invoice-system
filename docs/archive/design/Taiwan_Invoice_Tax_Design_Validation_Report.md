### 2. ✅ 三聯式 vs 二聯式發票區別

**驗證結果**: 通過 ✓

**驗證內容**:
根據搜尋結果，三聯式和二聯式發票區別如下：

| 項目 | 二聯式 (Bipart) | 三聯式 (Tripart) |
|------|-----------------|-----------------|
| 開立對象 | 個人、無統編機構 | 有統編的公司、行號 |
| 統一編號 | 不填 | 填寫客戶統編 |
| 聯數用途 | 存根聯(1) + 收執聯(2) | 存根聯(1) + 扣抵聯(2) + 收執聯(3) |
| 金額顯示 | 含稅金額 | 不含稅金額 |

**設計驗證**:
```
✅ 設計正確區分三聯式和二聯式
✅ 根據客戶統編自動選擇發票類型
✅ IsUnitPriceIncludeTax 正確標記
```

### 7. 🟡 發票月份與交易月份一致（實務規定）

**驗證結果**: 部分確認

**法規依據**:
- 《統一發票使用辦法》第9條 - 發票應記載「交易日期」

**發現的規定**:
搜尋結果中未直接提及「發票月份必須與交易月份完全一致」的強制規定，但存在以下相關規定：

1. 發票應據實載明交易日期（《統一發票使用辦法》第9條）
2. 交易日期決定申報期間（即按月份對應）

**設計中的假設**:
設計假設發票月份（開立月）與交易月份應一致，不一致時需要說明。

**實務考量**:
實務上存在以下合理情況：
- 跨月份交易（分期發票）
- 預開發票（需稅務機關核准）
- 修正發票（補開或作廢重開）

**建議**:
1. 保留此驗證邏輯
2. 允許月份不一致，但需要會計人員確認原因
3. 在稅務申報時特別標註
4. 不作為硬性規則，而是作為警告提示

**符合度**: 85%

### 8. ✅ "依第四章第二節規定"銷售額

**驗證結果**: 通過 ✓

**法規依據**:
- 《加值型及非加值型營業稅法》第四章第二節（非加值型營業稅計算）

**含義解釋**:
「依第四章第二節規定」指的是依據非加值型營業稅（Non-VAT）的計算方式進行稅務申報的營業人。根據法規，這類營業人銷售非經常買進、賣出的固定資產時，該項銷售額屬於免稅項目。
