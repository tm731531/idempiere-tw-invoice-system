## A. 財會高手需要檢查的項目

### A1. 三聯式vs二聯式發票的會計處理

**問題清單**：
- [ ] 三聯式（不含稅金額）和二聯式（含稅金額）在會計系統中應如何區分？
- [ ] 二聯式金額拆分成「不含稅 + 稅額」後，是否會影響應收帳款的帳務記錄？
- [ ] 三聯式回算稅額的會計處理方式是什麼？
  - 例：銷售額 1000 → 回算稅額 50（而非 1050 × 5% = 52.5）？
- [ ] 客戶統編是否必須記錄在發票上？會影響哪些會計帳戶？

**設計中的假設**：
- 三聯式金額 = 不含稅
- 二聯式金額 = 含稅，自動拆成兩部分
- 請驗證這個假設是否正確

- [ ] 進項稅折讓發生時，會計分錄應如何處理？

**設計中的假設**：
- 進項稅在 C_InvoiceLine 中用 `TW_TaxDeductible` 標記可否扣抵

## C. 系統設計驗證

### C1. 數據流驗證

**請檢查以下流程是否正確**：

```
銷售流程：
SO → 出貨 → 銷售發票
   ↓
   1. 自動選擇字軌（SALES_TRIPART 或 SALES_BIPART）
   2. 自動分配號碼（檢查連續性）
   3. 記錄開立日期（DateInvoiced）→ 決定申報期間
   4. 自動生成 QR Code
   5. 建立 TW_Invoice_Prefix_Map 記錄
   6. 更新 TW_TaxStatement 銷項稅
   7. 完成
```

採購流程：
```
PO → 收貨 → 採購發票
   ↓
   1. 手動輸入或掃描發票號
   2. 標記稅務屬性（可扣抵/不可扣抵）
   3. 標記稅務分類（BUSINESS/ENTERTAINMENT/MEAL等）
   4. 完成
   5. 如有折讓，建立 TW_InvoiceAdjustment
      - 強制當期申報
      - 系統提示「本折讓必須在 XX 期內申報」
```
